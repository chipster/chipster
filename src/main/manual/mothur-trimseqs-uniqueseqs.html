Metagenomics / Trim and filter reads with Mothur

Description

Trim and filter reads and remove duplicate reads.

Parameters

  • flip (yes, no) [no]
  • qaverage (0-40) []
  • qwindowaverage (Integer) []
  • qwindowsize (Integer) []
  • qstepsize (Integer) []
  • maxambig (0-10) []
  • maxhomop (0-50) []
  • minlength (0-1000) []
  • maxlength (0-1000) []
  • pdiffs (0-10) []
  • bdiffs (0-10) []

Details

The reads are checked against user-defined quality requirements. The output sequences will be trimmed to remove the user-provided primers, barcodes, and sequences that drop below a quality threshold. The reads are also checked for redundancy. If a quality file is provided, a trimmed, non-redundant quality file is produce to match the trimmed, non-redundant sequence file.

This tool is based on the Mothur package.

Output

The analysis output consists of the following:

  • reads.groups.txt: A two column list with the first column indicating the sequence names in the input file and the second column the group that it came from.
  • reads.trim.names.txt: A list of the selected sequences and the redundant sequences corresponding to each.
  • reads.trim.unique.fasta: A FASTA-formatted sequence file trimmed, non-redundant reads.
  • reads.trim.unique.qual: Quality values (if provided as input) for the trimmed, non-redundant reads
  • summary.trim.unique.tsv: Summary statistics for the sequences
